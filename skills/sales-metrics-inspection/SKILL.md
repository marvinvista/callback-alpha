# Sales Metrics Inspection

## Overview

Use this skill to turn sales metrics into a practical explanation of what is likely happening and what to do next.

The job is to find the loose thread, pull it, and keep going until the likely root cause is visible.

## When to Use

- Use when a rep, segment, or funnel is underperforming and metrics are available.
- Use when the user wants to distinguish activity problems from quality problems.
- Use when deciding where coaching or process fixes should focus.

## When Not to Use

- Do not use when the problem is messaging or product strategy without supporting metrics.
- Do not use when the user needs a live pipeline cleanup rather than a metric diagnosis.
- Do not use if there is no baseline or comparison set at all; note the data gap first.

## Inputs

- Metrics by rep, team, or time period
- Role context such as AE, SDR, or CS
- Historical baseline or peer comparison
- Any relevant stage, segment, or deal-size cuts

## Workflow

1. Clarify the scope:
   - rep
   - team
   - stage
   - time window
2. Review quantity metrics first.
3. Review quality metrics second.
4. Compare the current fingerprint against:
   - the same entity over time
   - peers
   - expected ranges
5. Pull the most suspicious metric thread.
6. Form a root-cause hypothesis:
   - activity shortfall
   - poor targeting
   - weak conversion quality
   - deal-size or pricing mix issue
   - stage-management issue
7. Recommend the next inspection layer:
   - call review
   - email review
   - process audit
   - coaching
   - hiring or territory change

## Output Contract

Return:

- `Observed metric changes`
- `Likely root cause`
- `Supporting evidence`
- `Alternative explanations`
- `Missing data`
- `Recommended actions`

## Quality Bar

- Distinguish quantity from quality.
- Avoid pretending a metric alone proves intent or skill.
- Name uncertainty when multiple explanations fit.
- Recommend a concrete next inspection step.
